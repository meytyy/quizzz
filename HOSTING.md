# Локальный хостинг для тестирования

## Быстрый старт

### Вариант 1: Development режим (для разработки)

```bash
# Запустить dev-сервер с доступом по сети
npm run dev:host
```

После запуска вы увидите:
```
➜  Local:   http://localhost:5173/
➜  Network: http://192.168.1.108:5173/
```

**Доступ с телефона:**
1. Убедитесь, что телефон подключен к той же Wi-Fi сети, что и компьютер
2. Откройте браузер на телефоне
3. Введите адрес: `http://192.168.1.108:5173/` (замените на ваш IP адрес)

**Как узнать свой IP адрес:**
- Windows: `ipconfig` → ищите IPv4 Address
- Mac/Linux: `ifconfig` или `ip addr`

### Вариант 2: Production режим (для финального тестирования)

```bash
# 1. Собрать production версию
npm run build

# 2. Запустить preview с доступом по сети
npm run preview:host
```

Это даст вам оптимизированную версию приложения, как она будет работать в продакшене.

## Доступ с разных устройств

### С компьютера:
- `http://localhost:5173/` - локальный доступ
- `http://192.168.1.108:5173/` - доступ по IP (ваш IP может отличаться)

### С телефона (в той же Wi-Fi сети):
- `http://192.168.1.108:5173/` - замените на ваш IP адрес

### С планшета:
- То же самое, что и с телефона

## Проверка подключения

1. **Проверьте, что устройства в одной сети:**
   - Компьютер и телефон должны быть подключены к одной Wi-Fi сети

2. **Проверьте firewall:**
   - Windows может блокировать доступ. Разрешите Vite в брандмауэре при запросе
   - Или откройте порт 5173 в настройках брандмауэра

3. **Проверьте IP адрес:**
   - Убедитесь, что используете правильный IP адрес (не localhost!)

## Решение проблем

### Не могу подключиться с телефона:

1. **Проверьте firewall:**
   ```bash
   # Windows: откройте "Брандмауэр Защитника Windows"
   # Разрешите входящие подключения для Node.js/Vite
   ```

2. **Проверьте, что сервер запущен с --host:**
   - Используйте `npm run dev:host` вместо `npm run dev`

3. **Проверьте IP адрес:**
   - Убедитесь, что используете правильный IP (не 127.0.0.1 или localhost)

4. **Попробуйте другой порт:**
   ```bash
   # В vite.config.ts можно изменить порт
   export default defineConfig({
     server: {
       host: true,
       port: 3000  // или другой порт
     }
   })
   ```

### Медленное подключение:

- Убедитесь, что оба устройства в одной сети
- Проверьте скорость Wi-Fi
- Попробуйте использовать кабель для компьютера

## Полезные команды

```bash
# Узнать IP адрес (Windows)
ipconfig

# Узнать IP адрес (Mac/Linux)
ifconfig
# или
ip addr

# Проверить доступность порта (Windows)
netstat -an | findstr 5173

# Проверить доступность порта (Mac/Linux)
lsof -i :5173
```

## Для постоянного хостинга

Если хотите сделать приложение доступным постоянно:

1. **Используйте ngrok** (для тестирования):
   ```bash
   npm install -g ngrok
   ngrok http 5173
   ```
   Это даст вам публичный URL, доступный из интернета

2. **Используйте Vercel/Netlify** (для продакшена):
   - Соберите проект: `npm run build`
   - Загрузите папку `dist` на хостинг

3. **Используйте свой сервер:**
   - Настройте nginx/apache
   - Укажите папку `dist` как корневую
